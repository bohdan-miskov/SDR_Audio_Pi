# SDR Audio Pi — Система Виявлення Цілей 📡

**SDR Audio Pi** — це серверна частина системи радіомоніторингу, розроблена на Python. Додаток аналізує частоти, порівнює їх з сигнатурами в БД та передає відповідні сигнали до клієнтського застосунку.

> **Архітектура:** Цей репозиторій містить логіку аналізу радіо- та звукових частот.

---

## 🎯 Основні можливості

- **Детекція цілей** — виявлення об'єктів за допомогою аналізу частоти в реальному часі
- **Мережева комунікація** — передача даних до UI додатку через TCP/IP
- **Локальна база даних** — даних про об'єкти для детектування (SQLite)

## ⚙️ Встановлення та Налаштування

### 0. Клонування репозиторію

```bash
git clone <url>
cd SDR_Pi
```

### 1. Створення віртуального оточення

```bash
# Створення (один раз)
python -m venv .venv

# Активація — Windows (Git Bash)
source ./.venv/Scripts/activate

# Активація — Linux / macOS
source .venv/bin/activate
```

### 2. Встановлення залежностей

Проект використовує `pip-tools` для управління залежностями та `Makefile` для спрощення команд.

```bash
# Встановлення pip-tools
pip install pip-tools

# Встановлення всіх залежностей (production + dev)
make deps
```

### Робота з реальним обладнанням

1. Переконайтеся, що додаток UI чатсини запущений та доступний у мережі
2. Запустіть лише головний застосунок:

```bash
source ./.venv/Scripts/activate
python main.py
```

## 🛠 Розробка та Контроль Якості

Усі налаштування лінтерів знаходяться у файлі `pyproject.toml`.

### Makefile — швидкі команди

| Команда      | Опис                                                                        |
| ------------ | --------------------------------------------------------------------------- |
| `make check` | Запуск усіх перевірок: Ruff (лінтер), MyPy (типізація), Deptry (залежності) |
| `make fix`   | Автоматичне виправлення стилістичних помилок та сортування імпортів         |
| `make deps`  | Компіляція та встановлення нових пакетів                                    |

> Введіть `make` у терміналі, щоб побачити повний список доступних команд.

### Pre-commit Hooks

Перед початком розробки ініціалізуйте локальний захист комітів:

```bash
pre-commit install
```

Це автоматично перевірятиме код перед кожним `git commit`.

> **⚠️ Екстрений пропуск (Bypass):** Якщо трапилася критична ситуація (гарячий фікс) і вам потрібно терміново зберегти коміт, проігнорувавши помилки лінтера, додайте прапорець `--no-verify` (або `-n`):
>
> ```bash
> git commit -m "Urgent hotfix" --no-verify
> ```

### CI/CD (GitHub Actions)

Налаштовано автоматичний пайплайн. При кожному пуші код проходить сувору перевірку — коміт буде відхилено у разі критичних синтаксичних помилок або проблем з імпортами.

### Додавання нових залежностей

Якщо вам потрібно додати нову бібліотеку:

1. Додайте її назву у `requirements.in` (production) або `requirements-dev.in` (dev-інструменти)
2. Виконайте:

```bash
make deps
```

## 🔧 База даних

Проект використовує **SQLite** для зберігання даних про об'єкти для можливості детектування:

- **Розробка:** локальна БД створюється через `populate_db_util.py`

## 🛠 Вирішення типових проблем

### `ModuleNotFoundError`

Переконайтеся, що ви запускаєте скрипти з кореневої папки проекту (де лежить `main.py`) і використовуєте прапорець `-m` для модулів:

```bash
# ✅ Правильно
python -m temp.populate_db_util

# ❌ Неправильно
python temp/populate_db_util.py
```

### Проблеми з БД

База даних — це SQLite. Якщо потрібно — видаліть файл БД і перезапустіть `populate_db_util`.

### Застосунок не запускається

```bash
# Перевірте, чи активовано віртуальне оточення
which python  # Має вказувати на .venv

# Синхронізуйте залежності
make sync
```
